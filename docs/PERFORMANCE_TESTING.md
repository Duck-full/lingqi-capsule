# 灵栖胶囊性能测试

## 测试目标

- Intel Mac 16GB，macOS 13.1。
- 冷启动首屏小于 1.5 秒。
- 常驻内存低于 150MB。
- 空闲 CPU 低于 2%。
- 常用交互响应低于 200ms。
- 压力数据为 365 天、5,000 条事项、每日 2,000 字灵感。

## 执行命令

数据层和文本分析压力测试：

```bash
cd work/DailyReminderWidget
./run_performance_test.sh
```

真实 App 常驻内存和 CPU 测试：

```bash
cd work/DailyReminderWidget
./build.sh
./run_app_memory_test.sh
```

测试数据写入隔离的临时用户目录，不会修改真实事项和灵感数据。结果输出到：

```text
outputs/performance/
```

## 优化策略

- 事项按日期建立内存索引，避免页面查询反复扫描全部事项。
- 提醒修改采用防抖后台写盘，单条交互不等待 JSON 持久化。
- 灵感改为按日文件存储，旧版 `daily-notes.json` 自动迁移且保留迁移完成标记。
- 关键词与情绪分析使用有上限的 `NSCache`。
- Intel 和低电量模式关闭持续 hover 位移、粒子动画及多层原生毛玻璃。
- Intel 最小化窗口时进入轻量渲染，Dock 恢复完成后再延迟恢复附加渐变和动画。
- Intel 首页玻璃卡片、状态球和小树苗关闭离屏阴影，降低 Retina 窗口缩放时的合成成本。
- 背景图片缓存限制数量和总成本，避免主题切换后长期占用解码内存。
- 系统通知只在事项增删改时增量更新，不再每次启动全量清空重建。
