# 灵栖胶囊知识画像与增强搜索 PRD

## 1. Summary

本版本升级个人知识库，新增知识画像分析和增强搜索能力。用户可以从历史灵感中看到知识增长趋势、类型占比、关键词关联，并通过多条件筛选快速定位可复用内容。

## 3. Background

当前知识库已经支持从历史灵感生成知识条目，也支持详情阅读、标签编辑、人工改分类和批量导出。随着记录数量增加，单纯列表会逐渐变得低效：用户需要理解自己长期沉淀了哪些方向的知识，也需要通过更精细的条件快速找到内容。

本版本不做复杂 AI 问答，不做可编辑知识图谱，不引入第三方重型图表库。优先把本地已有数据转成可解释、可搜索、可导出的知识画像。

## 4. Objective

目标：让用户理解“我最近在沉淀什么知识”，并能快速检索到相关内容。

### Key Results

- 用户可以查看知识增长趋势，至少支持按日、按周、按月三种粒度。
- 用户可以查看知识类型占比，至少展示分类条目数和字数占比。
- 用户可以查看关键词网络，首版最多展示 20 个节点和 40 条关系边。
- 用户可以组合使用关键词、分类、时间范围、情绪筛选。
- 筛选结果、画像统计、批量导出使用同一套筛选条件，避免结果不一致。
- 在 365 天、5000 条事项、每日 2000 字压力数据下，知识库页面保持可用，不出现明显卡顿。

## 5. Market Segment(s)

首版面向个人知识沉淀用户：

- 经常记录工作想法、项目复盘、产品设计笔记的人。
- 希望把零散灵感沉淀成可复用知识的人。
- 不想维护复杂知识库工具，但需要轻量统计和搜索的人。

约束：

- 本地优先，不依赖云端。
- 保持 macOS SwiftUI 原生体验。
- 不破坏现有知识详情、标签编辑、人工改分类、导出能力。
- Intel Mac 16GB、macOS 13.1 仍需可用。

## 6. Value Proposition(s)

用户收益：

- 从“记录很多内容”升级为“看懂自己积累了什么”。
- 通过趋势图看到近期知识积累是否持续。
- 通过类型占比知道自己更偏产品、工程、项目、生活还是成长。
- 通过关键词网络看到高频主题之间的关联。
- 通过组合筛选快速找到过去可复用的内容。
- 通过导出形成阶段性知识报告。

避免的问题：

- 历史内容越多越难找。
- 关键词只是标签，不能展示关系。
- 搜索只能单条件，定位效率低。
- 导出内容与当前筛选范围不一致。

## 7. Solution

### 7.1 UX / Prototype

知识库页面保留现有卡片风格，新增三个区域：

1. 知识画像概览区
   - 时间趋势卡片：展示条目数、字数随时间变化。
   - 类型占比卡片：展示各分类数量和字数占比。
   - 关键词网络卡片：展示关键词节点和共现关系。

2. 增强搜索区
   - 关键词输入。
   - 分类筛选。
   - 时间范围筛选：全部、近 7 天、近 30 天、近 90 天、自定义。
   - 情绪筛选：从已有 mood 数据聚合。
   - 清空筛选。

3. 知识列表区
   - 继续使用现有知识卡片。
   - 列表结果由统一筛选模型驱动。
   - 点击卡片进入现有详情页。

### 7.2 Key Features

#### P0

- 新增 `KnowledgeSearchFilter`，统一描述所有筛选条件。
- 增强 `KnowledgeBaseService.search`，支持组合筛选。
- 新增时间趋势数据模型与聚合能力。
- 新增类型占比数据模型与聚合能力。
- 在知识库页面展示趋势和类型占比。
- 批量导出使用当前筛选条件。

#### P1

- 新增关键词网络数据模型。
- 从同一条知识的关键词共现生成节点和边。
- 在知识库页面展示轻量关键词网络。
- 点击关键词节点可以反向填入筛选条件。
- 导出内容追加画像摘要。

#### P2

- 优化关键词网络布局。
- 增加画像总结文案。
- 增加图表细节动效，但保持克制。

### 7.3 Technology

- 核心数据模型和聚合逻辑放在 `Sources/KnowledgeBaseCore.swift`。
- 核心测试放在 `Tests/KnowledgeBaseCoreTests.swift`。
- SwiftUI 页面优先复用 `DailyReminderWidget.swift` 中已有知识库页面组件。
- 图表首版使用 SwiftUI 原生 `Path`、`Capsule`、`Grid`、`Canvas` 或基础布局实现，不引入第三方库。
- 大数据计算需要限制结果数量并尽量缓存，避免每次渲染重复计算。

### 7.4 Assumptions

- `KnowledgeEntry.date/category/keywords/mood/wordCount/sourceText` 足够支撑首版画像。
- 关键词网络只做“共现关系”，不做语义推理。
- 搜索增强优先保证准确和快，不优先做复杂动画。
- 自定义时间范围首版只在知识库页面内生效，不做全局设置持久化。

## 8. Release

## Acceptance Criteria

- 用户可以同时选择分类、时间范围、情绪和关键词。
- 筛选结果与知识列表、画像统计、导出结果一致。
- 时间趋势数据按日、周、月统计正确。
- 类型占比条目数和字数加总正确。
- 关键词网络最多展示 20 个节点、40 条边。
- 空数据状态清晰。
- 构建通过。
- 现有知识详情、标签编辑、人工改分类、批量导出不受影响。
