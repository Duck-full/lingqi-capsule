# 菜单栏快捷面板自适应布局设计

## 目标

修复菜单栏快捷面板在固定 `392 × 592` 尺寸下，上方状态栏和下方内容同时被裁切的问题，同时保持现有信息层级与交互。

## 根因

`MenuBarQuickPanelView` 使用固定高度容器承载一个自然高度超过 592 点的 `VStack`。SwiftUI 会将溢出的内容居中放置，导致顶部的 `HeaderStatusView` 与底部的品牌文案一起越过可视边界。

## 方案

1. 将面板高度调整为能够容纳常规完整内容的尺寸，并保留水平与垂直安全边距。
2. 让内容区域可以垂直滚动；当系统可用空间不足或内容随状态增长时，用户仍可访问全部操作与最近灵感。
3. 顶部状态栏放在固定的非滚动区域，始终完整可见；其余内容在下方单独滚动。
4. 不调整字号、卡片高度、文案和现有行为；只修改布局容器和尺寸约束。

## 验收标准

- 在默认面板尺寸下，顶部状态栏完整显示，不再被上边缘裁切。
- 最近灵感和底部品牌文案可访问，不再因内容溢出而丢失。
- 内容超过可视区时可滚动，顶部状态栏仍保持可见。
- 构建与现有核心测试通过。
